# CueForge Pre-Release Note Cadence

CueForge updates should feel alive, but not rushed or spammy.

## Default Timing

- T-6 hours: draft the pre-release note for approval.
- Release window: post only what actually passed the proof gate.
- After release: move missed items to the next update as priority work.

## Pre-Release Note Shape

Use relaxed player-facing wording:

- What is coming.
- What got tested.
- What still needs eyes.
- What changed because of tester feedback.
- What slipped and becomes priority next.
- What exact flow testers should try.

Avoid:

- Internal automation/tooling talk.
- Overclaiming results that did not pass.
- Same-copy posting across communities.
- Anything that sounds like a generic changelog.

## Approval Rule

Pre-release notes can be prepared automatically. Public posting still needs visible approval from the project owner.

## Rollover Rule

If a planned item misses the update window, it does not disappear. It becomes a named priority in the next pre-release note and release packet unless a higher-severity tester issue replaces it.
