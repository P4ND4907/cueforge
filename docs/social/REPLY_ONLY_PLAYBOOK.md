# CueForge Reply-Only Outreach Playbook

Last updated: May 22, 2026.

CueForge outreach is now comment-first. Do not make new public promo posts until the account has normal replies, profile trust, and a few real conversations.

## Mode

- Reply to existing discussions only.
- Save the thread in Community Hub -> Thread Memory before replying.
- Keep links out unless somebody asks for the app.
- Make every reply useful even if CueForge is never mentioned.
- Mark the saved thread as `commented`, `needs-followup`, or `filtered`.

## Good Replies

- Answer the actual question in the thread.
- Ask one useful follow-up if the setup is unclear.
- Mention testing method before tool name.
- Separate game issue, server issue, Discord issue, routing issue, mic gain, and EQ/tuning.
- Sound like a player, not a launch account.

## Skip

- Communities that say no AI-assisted posting.
- Threads where a link would look like a drive-by ad.
- Removed/filtered repost attempts.
- Arguments where the reply would not help the app or the person asking.

## Current Priority Queue

1. `r/buildapc` sound-card/DAC thread: no-link practical answer about clean output, routing, spatializer stacking, and one-match testing.
2. `r/EqualizerAPO` troubleshooting threads: help with endpoint selection, APO chain order, and avoiding stacked filters.
3. X replies only when somebody is already discussing game audio, headset setup, mic clipping, or FPS footsteps.
4. Discord remains the main hub for testers and real check-ins.
